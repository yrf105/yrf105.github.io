SQL命令速查

Posted on 2021-03-15 Edited on 2021-04-13

DISTINCT

关键词 DISTINCT 用于返回唯一不同的值

WHERE

提取满足指定条件的记录

  • operator: =, >, <, >= , <=, !=, <> 表示（不等于），可使用 not, and, or 连接
  • is null: 判空
  • between A and B: [A, B] 之间的值
  • in (a, b, c): 等于 a 或 b 或 c 的值
  • like 模糊查询
    • % 表示多个字值，**_** 下划线表示一个字符
    • M% : 查询以 M 开头的内容
    • %M% : 查询包含 M 的所有内容
    • %M_ : 查询 M 在倒数第二位的所有内容

ORDER BY

对结果集进行排序

  • 默认升序

LIMIT

限制 SELECT 返回的数据数量

JOIN

  • INNER JOIN：如果表中有至少一个匹配，则返回行
  • LEFT JOIN：即使右表中没有匹配，也从左表返回所有的行
  • RIGHT JOIN：即使左表中没有匹配，也从右表返回所有的行
  • FULL JOIN：只要其中一个表中存在匹配，则返回行
  • 通过 A.colname 可以访问到 A 表里的 colname 列

函数

聚合函数（输入多个数据，输出一个数据）

函数 说明
COUNT() 统计某一列的行数
SUM() 计算某一列的合计值，该列必须为数值类型
AVG() 计算某一列的平均值，该列必须为数值类型
MAX() 计算某一列的最大值
MIN() 计算某一列的最小值

当时用 group by 时，聚合函数的输入是每一个多数据的单元格。

max()

返回指定列的最大值

  • 有多个最大值会返回多个最大值

count()

COUNT(column_name) 函数返回指定列的值的数目（NULL 不计入）：

GROUP BY

GROUP BY 语句用于结合合计函数，根据一个或多个列对结果集进行分组。

having

在 SQL 中增加 HAVING 子句原因是，WHERE 关键字无法与合计函数一起使用。

注意：WHERE语句在GROUP BY语句之前，SQL会在分组之前计算WHERE语句。HAVING语句在GROUP BY语句之后，SQL会在分组之后计算HAVING语句。
