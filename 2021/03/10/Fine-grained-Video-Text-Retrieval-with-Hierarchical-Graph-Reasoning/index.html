Fine-grained Video-Text Retrieval with Hierarchical Graph Reasoning

Posted on 2021-03-10 Edited on 2021-03-11

摘要

通过学习联合嵌入空间来度量跨模态相似度的方法，不能很好的表示诸如：场景、目标、动作及其构成等，复杂的视觉和文本细节。本文提出 HGR （Hierarchical Graph Reasoning，分层图推理）模型。HGR 将图像-文本匹配分解为从全局到局部的多个层次（global-to-local levels）。具体来讲，HGR 将文本结构成层次化语义图，包括：事件（events）、动作（action）、实体（entities）以及各层间的关系。基于注意力机制的图推理被用来生成分层文本嵌入，从而指导学习各种不同层次的视频学习。

简介

keyword-based video retrieval → cross-modal video-text retrieval (①global vectors, ②sequential representations)

不管是全局向量还是序列化表示，都依然没有发挥出句子能够表达出结构化语义的优势。

  • 多层角色语义图
  • 基于注意力机制的图推理
  • 从相似句子中选定给出视频的正确匹配
  • 更喜欢完整的描述，而不是正确但不完整的描述

方法

Hierarchical Textual Encoding

global-to-local 结构非常适合视频描述的语义理解。

Semantic Role Graph Structure

通过 Simple BERT Models for Relation Extraction and Semantic Role Labeling 提供的 方法 构建语义角色图结构 (SRGS)。

Initial Graph Node Representation

将 SRGS 中的每个节点初始化为一个向量。

每一个视频描述 \(C\) 由 \(N\) 个单词组成 \(\{c_1, \cdots, c_N\}\)

对于 event 节点，使用 Bi-LSTM + attention mechanism 将节点初始化为向量 \(g_{e}\)

对于 action 和 entity 使用 Bi-LSTM + max pooling 得到向量组 \(g_{a} = \{g_{a,1},...,g_{a,N_{a}}\}\) 和 \(g_{o} = \{g_{o,1},...,g_{o,N_{o}}\}\), \(N_{a} N_{o}\) 分别表示 action 和 entity 节点的数量

Attention-based Graph Reasoning

简单使用 relational GCN 会导致参数暴增，效率低且可能过拟合。

本文方法： \[ g_i^0 = g_i \odot W_r r_{ij} \tag{6} \] \(g_i \in \{g_e, g_a, g_o\}\) ，\(g_i \in \mathbb{R}^{D \times 1}\)

\(W_r \in \mathbb{R}^{D \times K}\) 角色嵌入矩阵（每个节点独享）， \(D\) 是节点的维度，\(K\) 是节点的个数

\(g_i^0 \in \mathbb{R}^{D \times 1}\) GCN 的输入，\(g_i^l\) GCN 第 \(l\) 层节点 \(i\) 的输出

\(r_{ij} \in \mathbb{R}^{K \times 1}\) one-hot 向量，节点 \(i\) 和节点 \(j\) 之间的角色语义关系

\(\odot\) 带广播的逐元素相乘

使用 a graph attention network ，通过选择来自相邻节点的相关内容，来增强节点表示 \[ \tilde{\beta_{ij}} = (W_a^q g_i^l)^T(W_a^k g_i^l) / \sqrt{D} \tag{7} \]

\[ \beta_{ij} = \frac{exp(\tilde{\beta_{ij}})}{\sum_{j \in N_i}exp(\tilde{\beta_{ij}})} \tag{8} \]

\(N_i\) 节点 \(i\) 的邻居节点

\(W_a^q,W_a^k\) 计算图注意力的参数 \[ g_i^{l + 1} = g_i^l + W_t^{l +1} \sum_{j \in N_i}(\beta_{ij}g_j^l) \tag{9} \] \(W_t \in \mathbb{R}^{D \times D}\) 通用转换矩阵，为所有关系类型共享

将 GCN 第 \(L\) 层的输出作为最终的层次化文本表示，\(c_e\) 事件节点，\(c_a\) 动作节点，\(c_o\) 实体节点

这样将参数数量从 \(L \times K \times D \times D\) 降为 \(L \times D \times D + K \times D\)

Hierarchical Video Encoding

视频采用三层独立嵌入。

给定视频 \(V\) 由 \(\{f_1, \cdots, f_m\}\) 组成 \[ v_{x, i} = W_x^v f_i,\qquad x \in \{e,a,o\} \tag{11} \] 采用和 Eq(4) 相似的方法获得视频中的主要事件表示向量 \(v_e\)

对于 action ，\(v_a = \{v_{a,1}, \cdots, v_{a,M}\}\) ，对于 entity，\(v_o = \{ v_{o,1}, \cdots, v_{o, M} \}\) ，他们都是一系列逐帧的特征。这些特征只会与和他们在相同层次的文本特征进行匹配。

Video-Text Matching

通过汇总三个层级的结果得到整体的跨模态相似度。

Global Matching

\(s_e = \cos(v_e,c_e) \equiv \frac{v^Tc}{\Vert v \Vert \Vert c \Vert}\)

Local Attention Matching

\(s_{ij}^x = \cos(v_{x,j}, c_{x,i}) \qquad x \in \{a,o\}\) 这样的局部相似度隐含了对齐。但是缺乏适当的归一化。使用 stacked attention 归一化： \[ \varphi_{ij}^x = softmax(\lambda([s_{ij}^x]_+ / \sqrt{\sum_j [s_{ij}^x]_+^2})) \tag{12} \]

\[ [\cdot]_+ \equiv max(\cdot,0) \]

\[ s_{x,i} = \sum_j \varphi_{ij}^x s_{ij}^x \]

\[ s_x = \sum_i s_{x,i} \]

Training and Inference

最终的视频-文本相似度为 \[ s(v,c) = (s_e + s_a + s_o) / 3 \tag{13} \] 使用对比排名损失（contrastive ranking loss）训练模型 \[ L(v^+, c^+) = [\Delta + s(v^+,c^-) - s(v^+, c^+)]_+ \\ \qquad\qquad\quad+ [\Delta + s(v^-,c^+) - s(v^+, c^+)]_+ \tag{14} \] https://gombru.github.io/2019/04/03/ranking_loss/
