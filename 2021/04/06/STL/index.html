操作符重载和模板

操作符重载

https://en.cppreference.com/w/cpp/language/operators

限制：

  • 不能重载运算符 ::（作用域解析）、.（成员访问）、.*（通过成员指针的成员访问）及 ?:（三元条件）。
  • 不能创建新运算符，例如 **、<> 或 &|。
  • 不可能更改运算符的优先级、结合方向或操作数的数量。
  • 重载的运算符 -> 必须要么返回裸指针，要么（按引用或值）返回同样重载了运算符 -> 的对象。
  • 运算符 && 与 || 的重载失去短路求值。

分配器

operator new() 和 malloc()

operator new() 底层是 malloc()，存在额外开销（cookie 记录分配内存的大小以便 free 时不需要传入大小就可以释放内存），所以当为大量的小对象分配内存时会造成较大浪费。

allocators

容器默认的内存分配器是 allocator ，allocator 底层是 operator new() 和 operator delete()。

虽然 G2.9 用了如下的内存池，但是现在的 allocator 如上底层只是简单的使用 malloc()，原来变成了 __gnu_cxx::__pool_alloc。

容器，结构与分类

list

list = list + list_node + list_iterator

list

  • 带有头结点（前闭后开）的双向循环列表

list_node

  • list_node = pre_pointer + next_pointer + data

list_iterator

  • ++ ++(int) -- * ->

Iterator

  • iterator 的 5 中关联类型：value_type， pointer_type， reference_type， diff_type， category_type

  • 针对迭代器 traits 萃取机【使用对原生指针的偏特化】用于萃取非 class 类型迭代器（内置类型指针）的以上 5 中关联类型

  • 其他类型的 traits

vector

vector = start + finish + end_of_storage

  • 动态增长的数组
    • 二倍增长
    • 非原地扩充，需要重新找位置然后搬运数据
  • begin ，end， size， capacity， empty， []， front， back
  • push_back()
    • 借由 insert_aux （inset 也会使用到）实现，insert_aux 检查备用空间，若满则寻找新的两倍大小空间，将插入点之前的元素拷贝到新的空间，在新空间的插入点处构造插入元素，并将原空间中插入点之后的元素拷贝到新的空间。
    • 大量调用拷贝构造函数（新空间）和析构函数（原空间）
  • iterator
    • 所保存值类型的指针

array

  • 使得数组也可以融入到 STL 的大家庭。
  • std::array<int, 10> 必须指定大小
  • 指针就是迭代器

forward_list

单向列表，类似 list

deque

  • deque = start + finish + map(T**) + map_size
  • iterator = cur + first + last + node(T**)
    • 模拟连续空间，用户感受不到分段
    • ++（后 ++ 调用前 ++） --（后 -- 调用前 --） -
    • +=（+ 调用 += 实现，-= 调用 += 赋值实现），移动一个以上的位置，在一个缓冲区 ? 直接移动 :先移到正确的缓冲区再移动相应偏移量
    • [] 调用 + 实现
  • insert()
    • 可以向前推，可以向后推，deque 会选择元素移动量较小的那一种
  • 当 copy 控制中心时，是控制到新空间的中间部分

queue, stack

  • 他俩基于 deque 实现，所以又称为适配器（adaptor），也可以基于 list 实现；stack 可以用 vector 实现，queue 不能用 vector 实现（pop() 会调用 pop_front）；都不能用 set 和 map（编译会通过，因为编译器对 模板 是部分检查）；
  • 但是不许遍历，不提供 iterator

rb_tree

  • 高度适度平衡的二叉搜索树
  • 利于 search 和 insert
  • 中序遍历得到排序状态
  • 不应该改变节点的值
  • key 要唯一
  • 模板参数： key value keyofvalue compare alloc
    • keyofvalue – 如何从 value 中取出 data
    • compare – key 如何比较大小
  • node_count + __rb_tree_node* + key_compare
  • insert_unique() 插入唯一元素； insert_equal() 插入重复元素

set，multiset

  • key 就是 value，value 就是 key
  • 自动排序
  • 可遍历
  • 无法通过迭代器改变元素值
  • set 不可重复； multiset 可重复
  • 底层使用 rb_tree，（类比于deque 和 queue 的关系，set, multiset 也是一种 adaptor）

map，multimap

  • 自动排序
  • 可遍历
  • 无法通过迭代器改变 key，可以改变 value
  • map key 不可重复； multimap key 可重复
  • adaptor （类似 set, multiset 和红黑树的关系）
  • multimap 不能用 []
  • map 可以用 []， 当 key 不存在时会使用默认值创建并返回

hashtable

  • Separate Chaining – 解决冲突
  • 当元素个数大于 bucket 个数时，就将 bucket 增加两倍（不一定是两倍，而是两倍附近的质数，这些质数都是提前算好保存起来），然后再哈希
  • 模板参数：value，key，hashfunc，extractkey，equalkey，alloc
  • 成员变量：hashfunc，extractkey，equalkey，value，buckets，num_elements
  • 迭代器成员变量：cur（当前节点）hashtable（hashtable 本身）
