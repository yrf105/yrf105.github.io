6.824 Lab 1 MapReduce

Posted on 2021-04-10

https://pdos.csail.mit.edu/6.824/labs/lab-mr.html

简介

本实验你将建立一个 MapReduce 系统。你将实现 worker 进程来调用用户给定的 Map 和 Reduce 函数并处理文件读写，一个 coordinator 进程来分发任务给工作线程并处理失败的 workers。你将完成和 MapReduce 论文类似的事情。

你的工作

实验是实现一个分布式的 MapReduce， 由 coordinator 和 worker 组成。一个 coordinator 和一个或多个并发的 worker。真实系统中 worker 运行在一组不同的机器上，但是本实验中将运行他们在单个机器上。worker 和 coordinator 通过 RPC 进行通信。每个 worker 向 coordinator 请求任务，从一个或多个文件读取任务输入，执行任务，将任务输出写入一个或多个文件。coordinator 应该注意 worker 是否在规定的时间（本实验是 10 s）里没有完成任务并将未完成的任务给另一个 worker 去执行。

我们已经给出了初始代码。分别在 main/mrcoordinator.go 和 main/mrworker.go 中；不要修改这些文件。你应该将你的实现放在 mr/coordinator.go，mr/worker.go, mr/rpc.go。

规则

  • map 应该为 nReduce 个 reduce 任务将 intermediate keys 划分到 buckets 中，其中 nReduce 为 main/mrcoordinator.go 中传递给 MakeCoordinator() 的参数
  • worker 应该将第 X 个 reduce 任务的输出放到文件 mr-out-X 中
  • 一个 mr-out-X 文件的每个 Reduce 函数输出应包含一行。该行应以Go“％v％v” 格式生成，并使用键和值进行调用。在 main / mrsequential.go 中查看注释为“这是正确的格式”的行。如果您的实现偏离此格式太多，则测试脚本将失败。
  • 您可以修改 mr / worker.go，mr / coordinator.go 和 mr / rpc.go。您可以临时修改其他文件以进行测试，但是请确保您的代码可以与原始版本一起使用；我们将使用原始版本进行测试。
  • worker 应该将 intermediate Map 输出到当前目录的文件中，确保你的 worker 可以读取为 Reduce 任务读取他们。
  • main / mrcoordinator.go 期望 mr / coordinator.go 实现一个 Done() 方法，该方法在 MapReduce 作业完全完成时返回 true。届时，mrcoordinator.go 将退出。
  • 作业完全完成后， worker 进程应退出。一种简单的实现方法是使用 call（） 的返回值：如果 worker 程序未能与 coordinator 联系，则可以假定coordinator 由于完成工作而退出，因此 worker 也可以终止。根据您的设计，您可能还会发现 coordinator 给 worker 一个“请退出”的伪任务很有帮助。

提示

  • 一个简单的方法是修改 mr/worker.go 的 Worker() 向 coordinator 发送一个 RPC 请求任务。然后修改 coordinator 回应任务要处理的文件名作为 map 任务。然后修改 Worker 读取文件并调用应用程序 Map() 就像 mrsequential.go 那样。

  • 应用程序 Map 和 Reduce 函数在运行时使用 Go 插件包从名称以 .so 结尾的文件中加载。

  • 如果您在 mr / 目录中进行了任何更改，则可能必须重新构建您使用的所有 MapReduce 插件，例如 go build -race -buildmode = plugin ../mrapps/wc.go

  • 该实验依赖于 worker 共享文件系统。当所有 worker 都在同一台计算机上运行时，这很简单，但是如果 worker 在不同的计算机上运行，则需要像 GFS 这样的全局文件系统。

  • 中间文件的合理命名约定是 mr-X-Y，其中 X 是 Map 任务号，Y 是 reduce 任务号。

  • worker 的 map 任务代码需要将 intermediate key/value 存储在文件中，且能在 reduce 任务中正确读取。一个方法是使用 Go 的 encoding/json 包。将 k-v 写入 Json：
